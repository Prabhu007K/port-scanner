01

What is this tool?

An educational port scanner that discovers open TCP ports on a target host, estimates services, grabs banners, and maps risk — with live progress and exportable results. Built for learning network recon, not for unauthorized scanning.

TCP connect scan Banner grab UDP DNS demo Ethics-first
02

How port scanning works

A port is a numbered door on a host. Scanning probes each port to see if something accepts connections. Open ports reveal running services (web, SSH, databases) — useful for admins and attackers alike.

Connect scan (-sT)

socket.connect(host, port)
→ full TCP handshake
→ logged by target firewall

Implemented in this lab. Reliable, works without admin rights. Slower but honest for demos.

SYN scan (-sS) — stealth

Send SYN → receive SYN-ACK
→ send RST (never complete)
→ requires raw sockets / root

Explained only. Half-open scan is stealthier; this app falls back to connect scan if you select SYN.

03

What the scanner detects

🔌

Open ports

TCP connect with configurable timeout and threaded workers.

🏷

Service names

Port tables plus banner parsing (SSH, HTTP Server header, FTP).

📡

Banners

First bytes from services — version hints for defenders.

⚡

UDP / DNS

Optional UDP probe on port 53 — shows TCP-only scans miss things.

04

Legal & ethical use

  1. Only scan systems you ownOr have explicit written permission to test.
  2. Use safe demo targets127.0.0.1 (localhost) or scanme.nmap.org (Nmap's test host).
  3. Confirm consent checkboxRequired before scanning other targets in this app.
  4. Respect rate limitsMax 1024 ports per scan; adjustable timeout to reduce load.
